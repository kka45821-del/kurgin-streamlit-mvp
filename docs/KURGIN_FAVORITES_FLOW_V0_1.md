# KURGIN Favorites Flow v0.1

## 1. Назначение

Favorites v0.1 позволяет пользователю сохранить интересные камни в этом браузере.

Это безопасный локальный слой публичного MVP. Он нужен для удобства пользователя, чтобы он мог вернуться к интересным камням позже без регистрации и без backend.

## 2. Техническая модель

Favorites v0.1 работает как `localStorage only`.

Фиксированные ограничения:

- localStorage only;
- без регистрации;
- без backend;
- без синхронизации между устройствами;
- без гарантии вечного хранения;
- без записи в admin;
- без изменения `catalog.json`;
- без изменения `kurgin-data`;
- без изменения статусов камней.

Это значит, что избранное хранится только в браузере пользователя. Если пользователь очистит браузер, сменит устройство или откроет сайт в другом браузере, список может быть недоступен.

## 3. Data model

Минимальная структура записи в localStorage:

```json
{
  "stone_id": "61383",
  "added_at": "2026-05-26T20:45:00Z",
  "snapshot": {
    "shape": "Round",
    "carat": 2.46,
    "color": "D",
    "clarity": "VVS2",
    "score": 90.29,
    "priceText": "по запросу",
    "public_action": "request_price"
  }
}
```

Сохраняемые поля:

- `stone_id`;
- `added_at`;
- `snapshot.shape`;
- `snapshot.carat`;
- `snapshot.color`;
- `snapshot.clarity`;
- `snapshot.score`;
- `snapshot.priceText`;
- `snapshot.public_action`.

Актуальные данные должны браться из текущего `catalog.json`, если камень всё ещё присутствует в каталоге. Snapshot используется как fallback для случая, когда камень исчез из актуального каталога.

## 4. UX

Favorites v0.1 должен работать так:

- heart на карточке добавляет / убирает камень из избранного;
- detail sheet кнопка **«В избранное»** добавляет камень;
- Favorites tab показывает count избранных камней;
- Favorites page показывает список избранных камней;
- если камень есть в актуальном `catalog.json`, показываются актуальные данные;
- если камня уже нет в актуальном `catalog.json`, показывается текст:

> Камень больше не доступен в текущем каталоге.

Пустое состояние:

> Вы пока не добавили камни в избранное.

Допустимый уточняющий текст:

> Сохраняйте интересные камни из каталога, чтобы вернуться к ним позже.

## 5. Что Favorites v0.1 НЕ означает

Favorites v0.1 **не означает**:

- заказ;
- резерв;
- корзину;
- фиксацию цены;
- подтверждение наличия;
- `sold`;
- ownership transfer;
- checkout.

Добавление камня в избранное не меняет статус камня, не удерживает камень за пользователем и не создаёт обязательство со стороны KURGIN.

## 6. Safe text

Разрешённый публичный текст:

> Избранное помогает сохранить интересные камни в этом браузере. Оно не резервирует камни и не фиксирует цену.

Этот текст должен использоваться на Favorites page или рядом с пустым/информационным состоянием, если есть риск, что пользователь воспримет избранное как резерв.

## 7. Что нельзя делать сейчас

В рамках Favorites v0.1 нельзя:

- делать backend favorites;
- делать регистрацию;
- делать checkout;
- делать оплату;
- делать reserve;
- менять admin;
- трогать `kurgin-data`;
- публиковать `catalog.json`;
- менять статус камня;
- фиксировать цену;
- обещать наличие;
- синхронизировать избранное между устройствами.

## 8. Контрольное правило

```text
Favorite = local interest marker only.
```

Избранное — это только локальная отметка интереса пользователя. Любые заказ, резерв, price lock, checkout, payment, backend-account или синхронизация должны проектироваться отдельным scope.
