# KURGIN Favorites Release State v0.1

## 1. Что работает

Favorites v0.1 реализован и визуально проверен на live-сайте.

Подтверждённое рабочее состояние:

- localStorage favorites работает;
- heart на карточке добавляет камень в избранное;
- повторное нажатие heart удаляет камень из избранного;
- active heart state показывает, что камень уже добавлен;
- count badge на вкладке **Избранное** отображает количество избранных камней;
- Favorites page показывает список избранных камней;
- empty state работает;
- кнопка **«Удалить»** работает;
- кнопка **«Открыть карточку»** работает;
- snapshot fallback предусмотрен для случая, если камень исчез из актуального `catalog.json`;
- safe text отображается:

> Избранное помогает сохранить интересные камни в этом браузере. Оно не резервирует камни и не фиксирует цену.

## 2. Что Favorites v0.1 НЕ означает

Favorites v0.1 **не означает**:

- заказ;
- резерв;
- корзину;
- checkout;
- sold;
- price lock;
- подтверждение наличия;
- backend-сущность.

Избранное является только локальной отметкой интереса пользователя в браузере.

## 3. Что не затронуто

При реализации Favorites v0.1 не затрагивались:

- admin repo;
- `kurgin-data`;
- `catalog.json`;
- request-flow через MAX / Telegram / WhatsApp;
- pricing logic;
- checkout;
- оплата;
- роли;
- регистрация.

Favorites v0.1 не меняет цену, статус камня, наличие, публикацию или данные в админке.

## 4. Проверочный статус

Фактическая проверка:

- визуальная live-проверка пройдена;
- `0 ₽` не появился;
- `request_price` остался **«по запросу»**;
- checkout не появился;
- reserve не включился;
- GitHub Actions не запускались для экономии лимита.

Проверенные пользовательские действия:

```text
Каталог → heart → Избранное count
Избранное → список камней
Избранное → Открыть карточку
Избранное → Удалить
```

## 5. Текущая техническая модель

```text
Favorites v0.1 = localStorage only.
```

Ограничения:

- без backend;
- без регистрации;
- без синхронизации между устройствами;
- без гарантии вечного хранения;
- без изменения `catalog.json`;
- без записи в admin;
- без изменения статуса камня.

## 6. Следующий рекомендуемый слой

Следующий рациональный слой — вернуться к:

```text
Pricing Engine / первый priced batch
```

Не рекомендуется сразу переходить к корзине или checkout.

Причина: пока каталог работает как request-price / contact-flow. Перед корзиной или checkout нужно сначала безопасно провести хотя бы один priced batch через admin pricing confirmation и publication flow.

## 7. Что не делать следующим шагом без отдельного scope

Без отдельного утверждения не делать:

- реальный checkout;
- оплату;
- настоящий reserve;
- sold;
- backend favorites;
- регистрацию;
- price lock;
- order flow.

## 8. Контрольное правило

```text
Favorite = local interest marker only.
```

Favorites v0.1 помогает пользователю сохранить интересные камни в браузере, но не создаёт коммерческого или юридического действия.
